Quiero que construyas una aplicación web interna, profesional y fácil de usar para medir, analizar y estimar el desempeño del equipo UX de Simón Movilidad.

OBJETIVO GENERAL
Crear una herramienta web que sirva para 3 cosas al mismo tiempo:

1. Registrar el desempeño real del equipo UX por proyecto, producto, flujo o módulo.
2. Medir KPIs por rol usando producción, calidad, retrabajo, conexión, discovery y horas trabajadas.
3. Funcionar como un estimador evolutivo que, con cada proyecto nuevo registrado, mejore la capacidad de estimación futura en horas y esfuerzo por rol.

La herramienta debe servir para tomar decisiones de capacidad, planeación y carga de trabajo.

==================================================
ENFOQUE CLAVE DEL SISTEMA
==================================================
Este producto NO es solo un dashboard.
Debe funcionar como un sistema híbrido entre:
- registro histórico de desempeño
- calculadora de KPIs
- estimador de horas futuras por proyecto
- guía interna para entender KPIs y resultados

La lógica debe permitir:
- ingresar datos reales de proyectos ya ejecutados
- usar esos datos para estimar proyectos futuros
- comparar horas estimadas vs horas reales
- mejorar la precisión con el tiempo
- explicar claramente qué significa cada KPI
- explicar cómo interpretar cada resultado

4. KPIs A CALCULAR
==================================================
Todos los KPIs deben relacionarse también con horas.

A. Throughput por rol
throughput = vistas entregadas / horas de trabajo

B. Throughput ponderado
throughput_ponderado = (vistas principales + vistas responsive * factor + estados extra * factor) / horas trabajadas

Usa factores configurables:
- responsive = 0.75
- estado extra = 0.5
Estos valores deben poder editarse.

C. Velocidad por flujo
velocidad = total de vistas del flujo / tiempo total transcurrido

D. Tasa de retrabajo
retrabajo = vistas corregidas / vistas totales entregadas * 100

E. Índice de conexión
conexion = vistas conectadas correctamente / total vistas asignadas * 100

F. Puntaje de calidad
promedio de score stakeholder

G. Entrega a tiempo
entrega_a_tiempo = entregas a tiempo / entregas totales * 100

H. Eficiencia de investigación por rol
eficiencia_investigacion = horas de investigación / horas totales del rol * 100

I. Utilización productiva por rol
utilizacion = horas productivas / horas totales registradas * 100

J. Índice de revisiones
revisiones_promedio = total revisiones / total flujos

K. Cobertura de sistema de diseño
cobertura_ds = componentes reutilizados / componentes totales * 100

L. Capacidad efectiva por rol
capacidad_efectiva = (vistas entregadas ponderadas * (1 - retrabajo_decimal)) / horas trabajadas

M. Capacidad efectiva total
suma de capacidad efectiva de todos los roles participantes

N. Precisión de estimación
precision = 1 - abs(horas_reales - horas_estimadas) / horas_estimadas

Mostrar en porcentaje.

O. Desviación de estimación
desviacion = (horas_reales - horas_estimadas) / horas_estimadas * 100

P. Carga de investigación por proyecto
carga_investigacion = suma de horas de investigación de todos los roles

Q. Participación real por rol
participacion_rol = horas del rol / horas totales del proyecto * 100
