# UX Engagement Skill

## Propósito

Usa este skill cuando se necesite mejorar o evaluar la capacidad de una experiencia digital para:

- atraer atención de manera clara
- generar activación inicial
- sostener recurrencia útil
- fomentar hábitos saludables
- aprovechar gamificación con propósito
- aumentar retención sin caer en manipulación o adicción

## Cuándo usarlo

Activa este skill si la solicitud incluye alguno de estos objetivos:

- “quiero más engagement”
- “quiero que vuelvan a usar la app”
- “quiero mejorar retención”
- “quiero auditar onboarding”
- “quiero una experiencia más adictiva” → reinterpretar éticamente como “más atractiva y recurrente”
- “quiero usar gamificación”
- “quiero crear hábitos”
- “quiero mejorar activación”
- “quiero diseñar streaks, progreso, recompensas, retos o niveles”
- “quiero revisar si hay dark patterns”
- “quiero optimizar una landing, app o dashboard para recurrencia”

## Resultado esperado

Claude debe producir uno o varios de estos entregables:

1. Diagnóstico de engagement actual
2. Mapa del loop de hábito
3. Oportunidades de gamificación
4. Riesgos éticos y de manipulación
5. Recomendaciones priorizadas
6. Experimentos A/B o hipótesis de mejora
7. Copys o microinteracciones que fortalezcan motivación
8. Métricas sugeridas de activación, recurrencia y retención

## Marco conceptual que debes usar

### 1. Engagement útil

El engagement debe evaluarse como:

- frecuencia de retorno
- calidad de la interacción
- valor percibido
- claridad del beneficio
- motivación de continuar
- progreso visible
- sensación de control
- recompensa significativa

Nunca asumir que más tiempo en pantalla = mejor UX.

### 2. Formación de hábitos

Analiza la experiencia según este loop:

- Disparador: ¿qué inicia la acción?
- Motivación: ¿por qué actuar ahora?
- Capacidad: ¿es fácil hacerlo?
- Acción: ¿cuál es la conducta mínima?
- Recompensa: ¿qué obtiene la persona?
- Inversión: ¿qué deja listo para volver?

Debes observar si:

- el disparador es claro
- la acción principal es pequeña y comprensible
- la recompensa es inmediata o percibible
- existe progreso acumulativo
- la inversión genera retorno futuro

### 3. Gamificación

Evalúa o diseña elementos como:

- progreso visible
- niveles
- metas
- hitos
- badges
- retos
- streaks
- feedback inmediato
- recompensas variables moderadas
- colaboración
- estatus
- desbloqueos
- narrativa de avance
- mastery

Usa gamificación solo cuando:

- refuerce el valor real
- reduzca fricción
- aumente comprensión
- fortalezca constancia
- dé sensación de logro
- no distraiga del objetivo principal

### 4. Riesgo de adicción o manipulación

Siempre revisar si existen señales de:

- compulsión artificial
- presión por miedo a perder
- dependencia de recompensas vacías
- loops sin valor real
- notificaciones invasivas
- comparaciones sociales dañinas
- streaks castigadores
- monetización explotativa
- dificultad para salir, pausar o desactivar

Si aparecen, señalarlas y proponer alternativas éticas.

## Proceso de trabajo

### Modo 1: Creación / diseño

Cuando el usuario quiera diseñar una experiencia:

1. Identifica el comportamiento objetivo
2. Define el valor real para el usuario
3. Define el trigger principal
4. Diseña la acción mínima viable
5. Diseña feedback inmediato
6. Diseña progreso visible
7. Diseña recompensa significativa
8. Diseña inversión futura
9. Valida ética y bienestar digital
10. Propón métricas

### Modo 2: Auditoría

Cuando el usuario quiera auditar:

1. Identifica objetivo del producto
2. Identifica la conducta clave
3. Revisa onboarding
4. Revisa activación
5. Revisa retorno / recurrencia
6. Revisa motivadores y recompensas
7. Revisa claridad del progreso
8. Revisa riesgo de abandono
9. Revisa riesgo de dark patterns
10. Prioriza hallazgos

### Modo 3: Optimización / experimentación

Cuando el usuario quiera mejorar lo existente:

1. Detecta el cuello de botella
2. Relaciónalo con el loop de hábito
3. Identifica hipótesis
4. Propón test A/B
5. Define métricas de éxito
6. Define riesgo ético
7. Recomienda implementación incremental

## Principios rectores

- Diseñar para valor, no solo para retención
- Facilitar la conducta correcta
- Mostrar progreso temprano
- Recompensar con sentido
- Evitar sobrecarga cognitiva
- Dar autonomía y control
- Mantener transparencia
- No castigar al usuario por pausar
- No esconder salidas o configuraciones
- No sacrificar bienestar por negocio

## Formato de respuesta estándar

Cuando uses este skill, responde con esta estructura:

### 1. Objetivo de comportamiento

¿Qué conducta se quiere promover?

### 2. Diagnóstico actual

¿Qué funciona y qué frena el engagement?

### 3. Análisis del loop de hábito

- Trigger
- Motivación
- Capacidad
- Acción
- Recompensa
- Inversión

### 4. Gamificación aplicable

¿Qué mecánicas sí aportan valor y cuáles no?

### 5. Riesgos éticos

¿Qué elementos podrían volverse manipulativos o dañinos?

### 6. Recomendaciones priorizadas

Separadas en:

- alto impacto / baja complejidad
- alto impacto / alta complejidad
- quick wins de copy o UI

### 7. Experimentos sugeridos

Hipótesis, cambio, métrica esperada

### 8. Métricas

Ejemplos:

- tasa de activación
- tiempo a primer valor
- recurrencia semanal
- retención D1 / D7 / D30
- completion rate
- ratio de retorno
- uso repetido de la acción clave
- opt-out o desactivación de notificaciones

## Reglas de estilo

- Habla de engagement como resultado de valor + claridad + motivación
- No glorifiques la adicción
- Reencuadra “hacerlo adictivo” como “hacerlo valioso, memorable y recurrente”
- Si el usuario pide manipulación, redirige hacia diseño ético
- Sé concreto en patrones, ejemplos y mejoras visuales/copy
