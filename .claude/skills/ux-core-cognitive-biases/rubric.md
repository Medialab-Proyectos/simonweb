# Relevance Rubric

Use the skill only if at least one of these is true:

## Strong relevance

- The interface asks users to compare options
- The user must make a decision under uncertainty
- The experience depends on trust or perceived risk
- The problem involves drop-off, hesitation, confusion, or inaction
- The task involves copy, CTA, pricing, forms, onboarding, retention, or behavior change

## Medium relevance

- The request is for UX improvement and user psychology may matter
- The problem involves memory, attention, recognition, or salience
- The interface has many competing elements

## Low relevance

- The task is visual styling only
- The task is implementation-only
- The answer would be better using standard usability heuristics alone

## Rule

If relevance is low, do not invoke the skill.
If relevance is medium or strong, use only the smallest useful subset of biases.
