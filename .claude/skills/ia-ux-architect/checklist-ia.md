# Checklist de IA para creación y auditoría

## Organización

- [ ] El contenido está agrupado por lógica de uso
- [ ] No hay bloques redundantes
- [ ] No hay categorías ambiguas
- [ ] No se mezclan tareas distintas en el mismo lugar

## Etiquetado

- [ ] Los nombres usan lenguaje del usuario
- [ ] No hay dos nombres para la misma cosa
- [ ] No hay etiquetas genéricas vacías
- [ ] Las acciones se distinguen de los contenidos

## Navegación

- [ ] Existe una navegación principal clara
- [ ] Existe una navegación secundaria cuando hace falta
- [ ] El usuario sabe volver
- [ ] El siguiente paso es evidente
- [ ] No hay destinos huérfanos

## Jerarquía

- [ ] Lo importante aparece primero
- [ ] Las prioridades visuales reflejan prioridades funcionales
- [ ] No compiten demasiados elementos a la vez

## Búsqueda y filtros

- [ ] El volumen de información justifica búsqueda
- [ ] Los filtros son útiles y entendibles
- [ ] Los estados vacíos aportan guía
- [ ] La búsqueda tolera errores razonables

## Escalabilidad

- [ ] La taxonomía puede crecer
- [ ] La estructura soporta nuevos módulos
- [ ] No depende de excepciones arbitrarias

## UX

- [ ] La estructura reduce carga cognitiva
- [ ] No obliga a abrir múltiples pantallas para completar una tarea
- [ ] No obliga a memorizar información entre pasos
- [ ] Ayuda al usuario a entender el sistema
