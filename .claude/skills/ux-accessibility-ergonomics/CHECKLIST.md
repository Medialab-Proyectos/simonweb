# Checklist de Accesibilidad y Ergonomía

## 1. Legibilidad

- ¿La tipografía tiene tamaño suficiente para el contexto?
- ¿El contraste entre texto y fondo es adecuado?
- ¿La jerarquía visual ayuda a escanear?
- ¿La longitud de línea no fatiga?
- ¿El espaciado mejora lectura y comprensión?

## 2. Uso del color

- ¿La interfaz no depende solo del color para comunicar?
- ¿Error, éxito y advertencia tienen texto o iconografía de apoyo?
- ¿Los enlaces y botones se distinguen claramente?

## 3. Navegación e interacción

- ¿Se puede recorrer con teclado?
- ¿El foco es visible?
- ¿El orden del foco es lógico?
- ¿Existen estados claros para hover, focus, active, disabled?

## 4. Targets y esfuerzo físico

- ¿Los botones y controles tienen tamaño suficiente?
- ¿Hay espacio entre elementos táctiles?
- ¿Se evita tocar por error?
- ¿Las acciones frecuentes están en zonas cómodas?

## 5. Formularios

- ¿Todos los campos tienen label?
- ¿Las ayudas e instrucciones son visibles?
- ¿Los errores explican cómo corregir?
- ¿Se reduce la carga de escritura?
- ¿Se puede completar sin esfuerzo excesivo?

## 6. Carga cognitiva

- ¿La pantalla evita saturación?
- ¿Hay agrupación lógica?
- ¿El usuario entiende qué hacer primero?
- ¿Los CTA son claros?
- ¿La interfaz reduce dudas y fricción?

## 7. Movimiento y fatiga visual

- ¿Las animaciones son útiles y no excesivas?
- ¿No hay parpadeos molestos?
- ¿La saturación, brillo y densidad no cansan?
- ¿La experiencia soporta uso prolongado?

## 8. Robustez accesible

- ¿Los nombres de controles serían claros para un lector de pantalla?
- ¿La semántica es coherente?
- ¿Los componentes tienen estados accesibles?
- ¿Las tablas, modales y menús serían comprensibles para tecnología asistiva?
