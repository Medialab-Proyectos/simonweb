# Creative UI Landing Skill

## Propósito

Este skill ayuda a Claude Code a diseñar interfaces, landing pages y conceptos visuales de producto que no se limiten a patrones genéricos o plantillas comunes.

Su objetivo es generar propuestas:

- diferenciadas
- coherentes
- usables
- visualmente sólidas
- comercialmente claras
- soportadas en patrones de creatividad

No debe producir ideas “raras por ser raras”.  
Debe producir ideas novedosas pero útiles, entendibles y viables.

---

## Cuándo usar este skill

Usa este skill cuando el usuario pida:

- diseñar una landing page
- diseñar una interfaz web o mobile
- crear una propuesta visual diferenciada
- evitar resultados que parezcan plantilla
- conceptualizar una experiencia más original
- generar varias rutas creativas de UI
- auditar si una interfaz se ve demasiado genérica
- transformar un brief en una dirección creativa concreta

---

## Cuándo NO usarlo

No usar este skill si:

- el usuario solo necesita copiar exactamente un diseño existente
- la prioridad es compliance estricto sin exploración creativa
- el usuario solo quiere maquetación técnica sin exploración conceptual
- el encargo exige replicar un sistema visual cerrado sin cambios

En esos casos, usar un skill de implementación o design-system fidelity.

---

## Principio rector

Toda propuesta debe equilibrar estas 5 fuerzas:

1. Originalidad
2. Claridad
3. Usabilidad
4. Viabilidad
5. Coherencia visual

La creatividad nunca debe romper:

- comprensión del usuario
- jerarquía visual
- accesibilidad básica
- objetivos del negocio
- consistencia funcional

# Los 18 patrones de creatividad

## 1. Combinación remota

Combinar dos dominios conceptualmente lejanos para producir una dirección nueva.

Ejemplo:

- fintech + terapia
- movilidad + videojuego
- dashboard + storytelling editorial

## 2. Transferencia de dominio

Tomar una lógica, lenguaje o sistema de otra industria y traducirlo al producto actual.

Ejemplo:

- onboarding como entrenamiento deportivo
- pricing como menú degustación
- dashboard como misión táctica

## 3. Analogía estructural

Usar una estructura profunda como base conceptual del sistema.

Ejemplo:

- interfaz como mapa
- landing como documental
- panel como centro de mando

## 4. Inversión

Invertir el supuesto central del patrón habitual.

Ejemplo:

- en vez de “mostrar más”, ocultar y revelar progresivamente
- en vez de vender beneficio primero, mostrar riesgo evitado primero

## 5. Eliminación

Quitar un componente típico de la interfaz para forzar una mejor solución.

Ejemplo:

- quitar carruseles
- quitar exceso de CTA
- quitar navegación redundante

## 6. Sustitución

Cambiar un componente tradicional por otro más potente.

Ejemplo:

- cards por timeline
- tabs por narrativa scroll
- lista por mapa visual

## 7. Amplificación

Exagerar una propiedad clave para volverla memorables.

Ejemplo:

- contraste visual
- sensación premium
- velocidad percibida
- sensación de control

## 8. Reducción extrema

Reducir al mínimo para aumentar foco y elegancia.

Ejemplo:

- una sola promesa
- una sola acción principal
- una sola narrativa dominante

## 9. Cambio de escala

Mover el problema de nivel.

Ejemplo:

- de pantalla a flujo
- de flujo a ecosistema
- de feature a historia de valor

## 10. Fragmentación

Separar una experiencia compleja en bloques más legibles.

Ejemplo:

- problema
- evidencia
- solución
- confianza
- acción

## 11. Reorganización

Reordenar los elementos para cambiar la lectura y la percepción.

Ejemplo:

- credibilidad antes que features
- demo antes que pricing
- resultado antes que explicación

## 12. Reencuadre del problema

Cambiar la pregunta original del diseño.

Ejemplo:

- no “cómo se ve moderno”
- sino “cómo se vuelve inolvidable y claro para este mercado”

## 13. Emergencia

Diseñar reglas simples que generen una percepción rica.

Ejemplo:

- un sistema modular que se adapta por contexto
- secciones que cambian según tipo de usuario

## 14. Evolución iterativa

Generar varias mutaciones y seleccionar las mejores.

Ejemplo:

- explorar 3 rutas
- evaluar
- combinar
- refinar

## 15. Hibridación

Fusionar dos tipologías de producto o interfaz.

Ejemplo:

- landing + diagnóstico
- ecommerce + configurador
- dashboard + storytelling

## 16. Tensión creativa

Resolver una contradicción útil.

Ejemplo:

- premium pero cercano
- técnico pero simple
- emocional pero profesional

## 17. Violación controlada de normas

Romper una convención con intención y control.

Ejemplo:

- hero menos convencional
- asimetría calculada
- navegación no típica pero comprensible

## 18. Generación sistémica

Diseñar una lógica reusable, no solo una pantalla.

Ejemplo:

- sistema de secciones reutilizables
- patrones composables
- framework visual adaptable

# Proceso operativo obligatorio

Siempre seguir este flujo:

## Paso 1. Entender el encargo

Extraer:

- tipo de producto
- audiencia
- objetivo principal
- tono de marca
- restricciones técnicas
- competencia o referencias
- nivel deseado de diferenciación

## Paso 2. Detectar el riesgo de plantilla

Evaluar si la solución tendería a caer en:

- hero genérico
- cards repetidas
- CTA sin jerarquía
- layout SaaS clonado
- visuales intercambiables con cualquier marca
- copy predecible
- confianza superficial
- secciones sin narrativa

## Paso 3. Seleccionar patrones de creatividad

Elegir mínimo:

- 1 patrón de combinación
- 1 patrón de transformación
- 1 patrón de reestructuración
- 1 patrón de generación

Ideal:
aplicar de 4 a 6 patrones por propuesta.

## Paso 4. Generar 3 direcciones creativas

Crear siempre:

- Ruta A: segura pero diferenciada
- Ruta B: más atrevida
- Ruta C: híbrida o experimental controlada

## Paso 5. Filtrar por usabilidad y UI

Validar cada ruta con:

- jerarquía visual
- escaneabilidad
- consistencia
- accesibilidad
- claridad del CTA
- densidad visual
- ritmo entre secciones
- alineación con objetivo comercial

## Paso 6. Elegir y refinar

Seleccionar la mejor dirección o combinar dos.

## Paso 7. Entregar diseño accionable

La respuesta final debe incluir:

- concepto rector
- lógica creativa usada
- estructura de secciones
- dirección visual
- comportamiento UI
- componentes sugeridos
- riesgos
- recomendaciones

# Reglas para diseño de landing

Cuando el encargo sea una landing, Claude debe definir:

## 1. Tesis central

¿Qué idea domina toda la página?

## 2. Arquitectura narrativa

Orden recomendado:

- Hero
- dolor o tensión
- resultado o propuesta de valor
- prueba o confianza
- cómo funciona
- diferenciadores
- objeciones
- CTA final

Puede alterarse si la creatividad lo justifica.

## 3. Patrón creativo principal

La landing no debe mezclar demasiadas metáforas.  
Debe tener una dirección principal dominante.

## 4. Claridad comercial

Toda landing debe dejar claro:

- qué es
- para quién es
- por qué importa
- por qué confiar
- qué hacer después

## 5. Distinción visual

Evitar:

- SaaS hero genérico con dashboard flotante sin argumento
- degradados vacíos sin sistema
- exceso de badges
- 8 secciones iguales en cards
- mockups sin función narrativa

---

# Reglas para diseño de interfaces

Cuando el encargo sea una interfaz:

## 1. Diseñar por tareas, no por decoración

La creatividad debe mejorar:

- comprensión
- flujo
- feedback
- priorización
- control
- confianza

## 2. Diseñar estados

Siempre pensar:

- vacío
- cargando
- éxito
- error
- parcial
- bloqueado
- avanzado

## 3. Diseñar ritmo cognitivo

Regular:

- densidad
- agrupación
- secuencia
- contraste
- pausas visuales

## 4. Diseñar para diferenciación funcional

No solo cambiar colores o forma.
Cambiar:

- lógica de navegación
- presentación de la información
- ritmo de interacción
- modelo mental dominante

---

# Sistema de decisión creativa

Antes de dar una propuesta final, Claude debe responder internamente:

1. ¿Esto podría confundirse con una plantilla popular?
2. ¿La metáfora elegida realmente ayuda?
3. ¿La idea sorprende sin entorpecer?
4. ¿La narrativa visual acompaña el negocio?
5. ¿El usuario entiende qué hacer en menos de 5 segundos?
6. ¿La propuesta se siente propia de la marca?
7. ¿La diferenciación es superficial o estructural?

Si 3 o más respuestas son débiles, rehacer.

---

# Formato de salida obligatorio

Responder con esta estructura:

## 1. Diagnóstico

- qué tan genérico sería el enfoque tradicional
- dónde está la oportunidad creativa

## 2. Patrones aplicados

Listar los patrones elegidos y por qué

## 3. Concepto rector

Una frase clara

## 4. Dirección visual

- estilo
- composición
- ritmo
- color
- tipografía
- imagen o ilustración
- motion si aplica

## 5. Estructura propuesta

Pantallas o secciones en orden

## 6. Componentes clave

Qué componentes deben existir y por qué

## 7. Diferenciadores frente a plantilla

Qué hace esta propuesta distinta

## 8. Riesgos

Qué podría salir mal

## 9. Recomendaciones de refinamiento

Qué probar o iterar

# Modo auditoría

Si el usuario comparte una interfaz existente, evaluar:

- dónde cae en plantilla
- qué convenciones repite sin intención
- qué patrones creativos faltan
- cómo rediseñarla sin perder claridad

En modo auditoría, devolver:

- hallazgos
- oportunidades
- redirección conceptual
- mejoras concretas

---

# Modo exploración

Si el usuario pide ideas nuevas, devolver mínimo 3 conceptos:

- uno conservador
- uno intermedio
- uno disruptivo controlado

Cada concepto debe incluir:

- nombre
- idea base
- patrones aplicados
- estructura
- riesgo creativo

---

# Restricciones

Nunca:

- sacrificar accesibilidad por novedad
- generar layouts arbitrarios sin razón
- abusar de metáforas visuales
- producir experiencias confusas “porque son creativas”
- copiar sistemas famosos con cambios cosméticos
- depender de modas si no fortalecen el objetivo

---

# Criterio de calidad final

Una solución excelente debe sentirse:

- nueva pero comprensible
- distintiva pero usable
- estética pero funcional
- creativa pero estratégica
