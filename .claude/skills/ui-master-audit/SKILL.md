# UI Master Audit

Este skill está diseñado para apoyar tareas de diseño y auditoría de interfaces de usuario con foco en UI profesional.

## Cuándo usar este skill

Usa este skill cuando el usuario necesite:

- auditar una interfaz
- mejorar una pantalla
- rediseñar un dashboard
- evaluar color, contraste o jerarquía visual
- revisar consistencia de componentes
- diseñar una nueva vista
- comparar una pantalla contra sistemas de diseño reconocidos
- definir criterios visuales para un design system
- revisar formularios, tablas, modales o estados de feedback

## Enfoque

Este skill trabaja principalmente sobre:

- composición visual
- jerarquía
- color
- tipografía
- spacing
- layout
- iconografía
- componentes
- estados
- accesibilidad visual
- consistencia sistémica

No debe desviar el análisis hacia UX estratégico salvo cuando sea necesario explicar un problema de UI.

## Flujo de trabajo obligatorio

### 1. Entender la intención visual

Antes de evaluar o proponer, definir:

- propósito de la pantalla
- contenido principal
- acción primaria
- acción secundaria
- densidad esperada
- contexto de uso

### 2. Analizar estructura

Evaluar:

- grid
- alineación
- agrupación
- ritmo visual
- distribución
- saturación
- espacios internos y externos

### 3. Analizar lenguaje visual

Evaluar:

- paleta
- contraste
- jerarquía cromática
- tipografía
- pesos
- tamaños
- uso de bordes
- radio
- sombras
- elevación
- superficies
- fondos

### 4. Analizar componentes

Evaluar:

- botones
- inputs
- dropdowns
- tabs
- navegación
- cards
- tablas
- modales
- alertas
- tooltips
- badges
- mensajes de estado

### 5. Clasificar hallazgos

Usar:

- Crítico
- Medio
- Deseable

### 6. Recomendar con base

Toda recomendación debe:

- ser concreta
- explicar el porqué visual
- indicar el riesgo que resuelve
- priorizar el impacto
- mencionar benchmark si aplica

## Benchmarks posibles

Elegir uno o varios según el producto:

- Material Design 3
- Apple HIG
- Fluent UI
- Atlassian Design System
- Shopify Polaris
- Carbon Design System
- Ant Design

## Preguntas guía que siempre debe resolver

- ¿Qué ve primero el usuario?
- ¿La acción principal es inequívoca?
- ¿Hay suficiente contraste y legibilidad?
- ¿La pantalla respira o está saturada?
- ¿La interfaz se comporta como sistema?
- ¿Los estados interactivos están claros?
- ¿La tipografía construye jerarquía?
- ¿El color organiza o desordena?
- ¿La interfaz parece madura y consistente?

## Regla especial para auditoría

Cuando audites, debes separar:

- lo que funciona bien
- lo que genera fricción visual
- lo que rompe la consistencia
- lo que afecta comprensión
- lo que afecta confianza del producto

## Regla especial para diseño

Cuando diseñes, debes definir:

- la estructura de bloques
- la jerarquía visual
- la lógica cromática
- la escala tipográfica
- los componentes base
- los estados mínimos necesarios
- los riesgos de implementación

## Salidas permitidas

### A. Auditoría UI

- objetivo visual
- hallazgos positivos
- problemas detectados
- impacto visual o funcional
- recomendación concreta
- severidad
- benchmark de referencia

### B. Diseño UI

- objetivo de la pantalla
- estructura sugerida
- layout
- lógica de color
- jerarquía de texto
- componentes
- estados
- lineamientos de consistencia

### C. Benchmarking

- sistema comparado
- similitudes
- brechas
- oportunidades de mejora
- adaptación recomendada
