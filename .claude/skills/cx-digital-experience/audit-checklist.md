# Audit Checklist - CX Digital Experience

## A. Comprensión

- [ ] La propuesta de valor se entiende rápido
- [ ] El usuario sabe qué hacer primero
- [ ] Las acciones primarias están claras
- [ ] El lenguaje evita ambigüedad
- [ ] La jerarquía visual guía bien

## B. Esfuerzo

- [ ] El flujo evita pasos innecesarios
- [ ] No exige recordar información innecesariamente
- [ ] La interfaz no satura
- [ ] El contenido está fragmentado correctamente
- [ ] La interacción principal requiere poco esfuerzo mental

## C. Confianza

- [ ] Hay señales claras de seguridad o credibilidad
- [ ] Los estados del sistema son comprensibles
- [ ] Se informan consecuencias antes de confirmar
- [ ] Los errores no generan desconfianza
- [ ] La experiencia es consistente

## D. Toma de decisiones

- [ ] Las opciones importantes están jerarquizadas
- [ ] La interfaz ayuda a comparar
- [ ] No hay parálisis por exceso de opciones
- [ ] Existen defaults útiles
- [ ] Se reduce incertidumbre antes de decidir

## E. Emoción

- [ ] La interfaz reduce ansiedad
- [ ] Los mensajes acompañan con empatía
- [ ] La espera está bien gestionada
- [ ] La confirmación da tranquilidad
- [ ] El tono corresponde al momento del usuario

## F. Servicio

- [ ] La experiencia digital coincide con la realidad operativa
- [ ] Las expectativas están bien configuradas
- [ ] El soporte está disponible cuando se necesita
- [ ] El journey contempla recuperación ante fallos
- [ ] El usuario entiende qué pasará después

## G. Resultado

- [ ] El flujo ayuda al usuario a completar su objetivo
- [ ] La experiencia apoya el objetivo de negocio
- [ ] Hay oportunidades claras de mejora
- [ ] Los quick wins son identificables
- [ ] Se pueden medir los cambios propuestos
