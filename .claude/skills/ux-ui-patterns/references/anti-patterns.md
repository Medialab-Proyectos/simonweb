# Anti-patterns

- CTA principal compitiendo con múltiples acciones secundarias
- usar carrusel para mensajes clave
- esconder navegación primaria en lugares poco visibles
- formularios largos sin progreso ni guardado
- errores solo al enviar el formulario
- placeholder usado como label
- íconos sin texto cuando la acción no es obvia
- tablas densas en móvil sin estrategia responsive
- modales para tareas largas o de análisis
- acordeones cuando el usuario necesita comparar contenidos
- scroll infinito en tareas de búsqueda analítica
- estados vacíos sin instrucción
- dependencia excesiva de color para estados
- confirmaciones ambiguas
- usar disabled sin explicar por qué
- cambios de patrón entre pantallas equivalentes
