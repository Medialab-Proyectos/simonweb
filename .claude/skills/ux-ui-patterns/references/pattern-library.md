# Pattern Library

## 1. Navegación

- Top navigation: útil en desktop con pocas categorías principales.
- Sidebar: útil para productos complejos o con alta densidad funcional.
- Bottom navigation: útil en móvil para 3 a 5 destinos principales.
- Tabs: útil para cambiar entre vistas hermanas de forma rápida.
- Breadcrumbs: útil en jerarquías profundas.
- Stepper / wizard: útil en procesos secuenciales.
- Mega menu: útil en ecosistemas con muchas categorías.

## 2. Descubrimiento y búsqueda

- Search bar: cuando el usuario ya sabe qué busca.
- Autocomplete: reduce escritura y errores.
- Faceted filters: útil en catálogos grandes.
- Sort: útil para reordenar por prioridad.
- Recent / suggested / popular: acelera exploración.

## 3. Formularios y captura

- Single-column form: mejora lectura y completitud.
- Inline validation: reduce errores tardíos.
- Input masks: útil en formatos estructurados.
- Smart defaults: acelera tareas repetitivas.
- Multi-step form: útil cuando hay muchos campos o bloques distintos.
- Save progress: esencial en tareas largas.

## 4. Feedback y estado

- Skeleton loading: mejor que spinner cuando carga contenido estructurado.
- Toast: útil para confirmaciones no críticas.
- Inline error: útil para errores de campo.
- Success state: confirma resultado.
- Empty state: enseña qué hacer cuando no hay datos.
- Progress indicator: útil en tareas largas.

## 5. Visualización de información

- Cards: contenido modular, exploratorio o heterogéneo.
- Table: comparación precisa entre filas y columnas.
- Accordion: compresión vertical cuando no se necesita comparar.
- Dashboard widgets: monitoreo resumido.
- Timeline: secuencia temporal.
- Comparison matrix: elección entre opciones.

## 6. Onboarding

- Guided tour: útil si el producto es complejo, pero debe ser breve.
- Checklist onboarding: útil para activación gradual.
- Progressive onboarding: enseña en contexto.
- Empty state educativo: ideal para primeras veces.

## 7. Conversión

- Sticky CTA: mantiene acción visible.
- Pricing comparison: ayuda decisión entre planes.
- Social proof: refuerza confianza.
- Risk reduction: refuerza seguridad antes de convertir.

## 8. Confianza

- Trust badges: solo si tienen respaldo real.
- Testimonios: útiles cerca de puntos de decisión.
- Transparencia de costos: reduce fricción.
- Confirmaciones claras: mejoran sensación de control.

## 9. Accesibilidad transversal

- labels visibles
- foco visible
- contraste suficiente
- targets táctiles adecuados
- estados no dependientes solo del color
- navegación por teclado
- mensajes de error comprensibles
