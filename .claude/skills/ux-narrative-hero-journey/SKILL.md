# UX Narrative Hero Journey Skill

## Propósito

Este skill ayuda a Claude Code a aplicar principios de:

- narrativa aristotélica
- ruta del héroe
- estructura dramática
- progresión emocional
- construcción de sentido
- transformación del usuario

para:

- diseñar productos digitales
- mejorar flujos UX
- estructurar onboarding
- definir journeys
- crear pantallas con progresión narrativa
- auditar si una experiencia tiene claridad, tensión, avance, recompensa y transformación

---

## Cuándo usar este skill

Usa este skill cuando el objetivo sea:

1. Diseñar una experiencia que genere conexión emocional.
2. Transformar un flujo frío o técnico en uno más humano y memorable.
3. Crear onboarding, tutoriales, journeys o experiencias guiadas.
4. Auditar si un producto acompaña al usuario como protagonista de una historia.
5. Mejorar engagement, progresión, sentido de avance y recompensa.
6. Construir una experiencia donde el usuario pase de un estado inicial a uno transformado.
7. Diseñar experiencias educativas, financieras, de salud, bienestar, productividad, community o gamificadas.
8. Redactar microcopys, títulos, empty states, tooltips o mensajes con progresión narrativa.
9. Evaluar si la experiencia tiene conflicto claro, guía, reto, apoyo, logro y resolución.
10. Convertir requerimientos funcionales en experiencias con arco narrativo.

---

## Cuándo NO usarlo como enfoque principal

No usar este skill como marco principal cuando:

- el flujo sea puramente operativo y ultra corto
- el usuario necesite máxima velocidad sin carga narrativa
- el contexto exija neutralidad extrema
- la narrativa pueda distraer de tareas críticas
- la prioridad sea cumplimiento normativo puro sin capa emocional

En esos casos, este skill puede usarse como capa secundaria, no como estructura dominante.

## Principios base del skill

### 1. Narrativa aristotélica aplicada a UX

Toda experiencia debe poder mapearse en:

- inicio
- desarrollo
- clímax
- resolución

Interpretación UX:

- **Inicio:** contexto, orientación, expectativa, propósito
- **Desarrollo:** fricción útil, progresión, decisiones, aprendizaje
- **Clímax:** acción clave, punto de verdad, confirmación crítica
- **Resolución:** cierre, recompensa, claridad, siguiente paso

### 2. Usuario como protagonista

El usuario no es espectador del producto; es el protagonista.
El producto:

- no debe robar protagonismo
- debe actuar como guía, sistema o mentor
- debe reducir ambigüedad
- debe acompañar la transformación

### 3. Ruta del héroe aplicada a UX

La experiencia puede estructurarse en etapas narrativas equivalentes a:

1. Mundo ordinario
2. Llamado a la aventura
3. Duda o resistencia
4. Encuentro con guía
5. Cruce del umbral
6. Pruebas, aliados y obstáculos
7. Aproximación al reto central
8. Prueba decisiva
9. Recompensa
10. Camino de regreso
11. Transformación
12. Regreso con el logro

Estas etapas no siempre aparecen completas, pero la experiencia debe contener una progresión equivalente.

### 4. Toda experiencia debe producir cambio

Una buena UX narrativa logra al menos uno de estos cambios:

- mayor claridad
- menor ansiedad
- mayor confianza
- sensación de avance
- dominio progresivo
- percepción de logro
- identidad reforzada
- acción completada con sentido

### 5. La narrativa debe servir a la usabilidad

La historia nunca debe:

- ocultar información crítica
- dificultar la comprensión
- introducir metáforas confusas
- retrasar tareas importantes
- sacrificar accesibilidad o claridad

La narrativa debe reforzar la experiencia, no entorpecerla.

## Modelo operativo del skill

Cuando uses este skill, siempre analiza o diseña la experiencia con estas capas:

### Capa A. Estado inicial del usuario

Definir:

- quién es el usuario
- qué sabe al entrar
- qué siente
- qué quiere lograr
- qué teme
- qué lo frena
- cuál es su contexto real

### Capa B. Conflicto o tensión

Definir:

- cuál es el problema central
- qué incertidumbre existe
- qué decisión o reto debe enfrentar
- qué riesgo percibe
- qué fricciones internas o externas aparecen

### Capa C. Guía o sistema de apoyo

Definir:

- cómo el producto orienta
- qué señales reduce ansiedad
- qué feedback da
- qué explica
- qué valida
- cómo acompaña sin sobrecargar

### Capa D. Progresión

Definir:

- cómo siente avance el usuario
- qué hitos encuentra
- cómo cambia el nivel de dificultad
- qué recompensas recibe
- cómo se mantiene motivado

### Capa E. Momento decisivo

Definir:

- cuál es el punto de mayor relevancia
- qué decisión o acción central ocurre
- qué debe sentirse claro y seguro
- qué necesita el usuario justo antes de actuar

### Capa F. Resolución y transformación

Definir:

- cómo termina la experiencia
- qué ganó el usuario
- qué aprendió
- qué cambió
- cuál es el siguiente paso natural

## Instrucciones para diseño UX

Cuando se solicite diseñar una experiencia, Claude debe:

1. Identificar el protagonista
2. Describir el estado inicial emocional y funcional
3. Definir el objetivo narrativo del flujo
4. Mapear el journey usando narrativa aristotélica
5. Mapear el journey usando ruta del héroe
6. Proponer estructura de pantallas, pasos o momentos
7. Diseñar interacciones que reduzcan fricción emocional
8. Proponer mensajes y microcopys alineados con la etapa narrativa
9. Garantizar claridad, accesibilidad y usabilidad
10. Explicar cómo la experiencia transforma al usuario

### Entregables esperados en modo diseño

Según el pedido, entregar uno o varios de estos:

- narrativa del journey
- user flow
- mapa de pantallas
- onboarding
- estructura de microcopy
- criterios UX/UI
- recomendaciones de engagement
- oportunidades emocionales
- momentos de feedback y recompensa
- propuesta de auditoría preventiva

## Instrucciones para auditoría UX

Cuando se solicite auditar una experiencia, Claude debe evaluar:

### 1. Inicio

- ¿La experiencia contextualiza?
- ¿Explica qué está pasando?
- ¿Da propósito claro?
- ¿Reduce ansiedad inicial?
- ¿Establece expectativa realista?

### 2. Desarrollo

- ¿Existe progresión clara?
- ¿El usuario entiende en qué parte va?
- ¿Las pruebas o decisiones tienen sentido?
- ¿Hay apoyo contextual?
- ¿La experiencia mantiene motivación?

### 3. Clímax

- ¿La acción más importante está bien preparada?
- ¿Hay claridad antes del compromiso?
- ¿El usuario tiene confianza para actuar?
- ¿Se explican consecuencias, beneficios o riesgos?

### 4. Resolución

- ¿El cierre es claro?
- ¿Hay sensación de logro?
- ¿Se refuerza el avance?
- ¿Se define el siguiente paso?
- ¿Hay transformación perceptible?

### 5. Ruta del héroe

- ¿El producto reconoce el estado inicial del usuario?
- ¿Existe un llamado claro?
- ¿Se acompaña la resistencia o duda?
- ¿El sistema actúa como guía?
- ¿Hay umbral claro al empezar?
- ¿Se perciben pruebas y logros?
- ¿Existe recompensa?
- ¿Se evidencia transformación?

### 6. Riesgos de mala narrativa

- exceso de drama
- metáforas forzadas
- tono infantil o artificial
- progreso irreal
- clímax sin preparación
- recompensa vacía
- lenguaje emocional sin soporte funcional
- storytelling que tapa problemas de UX

## Reglas de calidad

Claude debe cumplir siempre lo siguiente:

1. No romantizar el sufrimiento del usuario.
2. No crear fricción innecesaria solo por “hacer historia”.
3. No usar metáforas oscuras si afectan comprensión.
4. No reemplazar claridad funcional por lenguaje poético.
5. Diseñar primero para comprensión, luego para emoción.
6. Mantener consistencia entre tono, interfaz y momento del flujo.
7. Asegurar que la narrativa fortalezca la accesibilidad.
8. Traducir siempre lo narrativo en decisiones concretas de UX.
9. Vincular el storytelling con objetivos reales del producto.
10. Priorizar confianza, claridad y avance perceptible.

---

## Heurísticas de evaluación narrativa UX

Usa estas preguntas como checklist:

- ¿El usuario entiende dónde está y hacia dónde va?
- ¿La experiencia tiene una promesa clara?
- ¿El producto reconoce el miedo o duda del usuario?
- ¿Hay una transición clara del “antes” al “después”?
- ¿Se percibe progreso paso a paso?
- ¿Hay momentos de validación?
- ¿El punto más importante del flujo está bien preparado?
- ¿El cierre deja claridad y sensación de logro?
- ¿La experiencia cambia cómo el usuario se siente?
- ¿La narrativa está al servicio de la tarea?

Siempre que este skill esté activo, interpreta cualquier solicitud relacionada con UX usando esta lógica:

"Analiza o diseña la experiencia como una transformación del usuario. Identifica estado inicial, conflicto, guía, progresión, punto decisivo y resolución. Usa narrativa aristotélica y la ruta del héroe como marcos de apoyo, sin sacrificar claridad, usabilidad, accesibilidad ni precisión funcional. Convierte el análisis narrativo en decisiones concretas de UX, UI, contenido, flujo y auditoría."

- Diseña un onboarding para una app financiera usando ruta del héroe.
