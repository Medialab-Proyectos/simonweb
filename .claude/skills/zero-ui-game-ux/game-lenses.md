# Lentes de teoría de juegos para UX

## Objetivo

Usar teoría de juegos para entender una experiencia como una interacción estratégica entre actores con intereses no siempre alineados.

## Modelos prácticos para UX

### 1. Juego de confianza

Úsalo cuando:

- el usuario entrega datos
- debe pagar
- delega una decisión a IA
- acepta una recomendación del sistema

Preguntas:

- ¿qué debe creer el usuario para avanzar?
- ¿qué puede hacer el sistema para merecer esa creencia?
- ¿qué pasa si el usuario sospecha oportunismo?

### 2. Juego de información asimétrica

Úsalo cuando:

- el negocio o la IA saben más que el usuario
- hay letra pequeña
- hay scoring, ranking o recomendaciones
- hay decisiones algorítmicas

Preguntas:

- ¿qué información falta para una decisión libre?
- ¿qué explicaciones deberían hacerse visibles?
- ¿cómo reducir la asimetría sin sobrecargar?

### 3. Juego de compromiso progresivo

Úsalo cuando:

- onboarding
- formularios largos
- ventas complejas
- journeys por etapas

Preguntas:

- ¿cada paso aumenta claridad o solo hunde más al usuario?
- ¿hay puntos de salida digna?
- ¿el compromiso es voluntario o por fricción de escape?

### 4. Juego principal-agente

Úsalo cuando:

- un agente de IA actúa por el usuario
- un sistema recomienda o ejecuta
- hay delegación

Preguntas:

- ¿el agente persigue el objetivo del usuario o el del sistema?
- ¿qué guardrails necesita?
- ¿qué acciones requieren aprobación humana?

### 5. Juego cooperativo

Úsalo cuando:

- soporte + producto + usuario deben colaborar
- hay varios roles
- existen procesos B2B, administrativos o regulados

Preguntas:

- ¿qué incentivos impiden la cooperación?
- ¿qué información compartida facilita coordinación?
- ¿cómo evitar culpas cruzadas?

## Variables mínimas

Para cada actor identifica:

- meta
- miedo
- incentivo
- poder
- información
- costo
- estrategia probable

## Salida esperada

Concluye siempre:

- equilibrio actual
- equilibrio deseado
- cambio de diseño necesario para mover el sistema
